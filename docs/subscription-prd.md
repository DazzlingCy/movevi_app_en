# MOVEVI Premium 订阅功能 PRD

## 1. 功能目标

MOVEVI Premium 是应用内月度会员功能。用户订阅后，可以解锁全部 Premium 城市路线，并在会员有效期内进入路线详情、开始跑步和获得完整路线体验。

当前版本为前端 mock 订阅，不接入真实支付、后端账号体系、App Store、Google Play 或 Stripe。功能重点是跑通用户订阅、首月优惠、权益解锁、取消续订、恢复续订、支付异常处理和本地状态保存。

## 2. 本期范围

本期支持：

- 会员套餐：Premium Monthly。
- 常规价格：$4.99/month。
- 首月优惠：首次订阅用户首月 $0.99。
- 会员权益：解锁全部 Premium 城市路线。
- 订阅周期：30 天。
- 支付方式：mock 信用卡、mock Apple / Google Pay。
- 本地保存订阅状态，刷新页面后保留有效订阅。
- 用户可订阅、取消续订、恢复续订、更新支付方式。

## 3. 用户使用流程

### 3.1 免费用户进入订阅

用户可以从以下入口进入订阅页：

- Profile 页 Premium 卡片。
- 未订阅时点击 Premium 路线。
- 未订阅时点击路线详情页的 Unlock with Premium。

进入订阅页后，用户看到：

- MOVEVI Premium 套餐名称。
- 首月优惠价或常规月费。
- 会员权益说明。
- 自动续订说明。
- 取消说明。
- 订阅按钮。

### 3.2 首次订阅用户

首次订阅用户可使用首月优惠。

展示规则：

- 用户未使用过首月优惠时，订阅入口展示首月 $0.99。
- 支付页展示本次应付 $0.99。
- 首月结束后按 $4.99/month 续订。
- 用户成功订阅后，系统标记该用户已使用过首月优惠。

订阅成功后：

- 状态变为 Active。
- 当前周期结束日为订阅成功后的 30 天。
- 记录付款方式。
- 账单记录展示首月优惠金额。
- Premium 路线立即解锁。

### 3.3 非首次订阅用户

如果用户已经使用过首月优惠，后续重新订阅时不再展示 $0.99。

展示规则：

- 订阅入口展示 $4.99/month。
- 支付页展示本次应付 $4.99。
- 成功后进入 Active。
- Premium 路线立即解锁。

### 3.4 信用卡支付流程

1. 用户选择 Credit Card。
2. 用户输入持卡人姓名、卡号、有效期和 CVC。
3. 页面校验输入是否合法。
4. 校验失败时，在对应字段下展示错误提示。
5. 校验通过后进入 mock 支付处理。
6. 支付成功后开通 Premium。

校验要求：

- 姓名不能为空。
- 卡号必须是合法卡号。
- 有效期必须是未来日期。
- CVC 必须是 3-4 位数字。

### 3.5 钱包支付流程

1. 用户选择 Apple / Google Pay。
2. 用户点击 Verify wallet。
3. 钱包验证成功后，按钮状态变为 Ready to pay。
4. 用户提交支付。
5. 支付成功后开通 Premium。

如果用户未完成钱包验证就提交，应提示先验证钱包。

### 3.6 已订阅用户查看会员

已订阅用户进入订阅页后看到：

- 当前会员状态。
- 套餐名称。
- 续订日期。
- 当前付款方式。
- Manage payment 入口。
- Cancel membership 入口。

Profile 页展示会员摘要，告诉用户 Premium 当前可用。

### 3.7 已订阅用户更新支付方式

用户点击 Manage payment 后进入支付页。

更新成功后：

- 会员状态保持 Active。
- 付款方式更新为新的卡片或钱包信息。
- 如果之前有支付异常提示，应清除异常提示。
- 本次 mock 更新不产生真实扣费。

### 3.8 用户取消续订

用户点击 Cancel membership 后，需要看到二次确认弹窗。

确认取消后：

- 状态变为 Canceled at period end。
- 自动续订关闭。
- Premium 权益不会立即关闭。
- 用户仍可使用 Premium 到当前周期结束日。
- 页面展示会员结束日期。
- 页面展示 Restart membership 入口。

取消续订不是退款，也不是立即关闭会员。

### 3.9 用户恢复续订

当用户处于 Canceled at period end 且当前周期未结束时，可以点击 Restart membership。

恢复成功后：

- 状态回到 Active。
- 自动续订重新开启。
- Premium 权益继续可用。
- 页面不再展示即将结束提示。

### 3.10 订阅过期后重新订阅

当前周期结束后，如果用户没有恢复续订：

- 状态变为 Expired。
- Premium 权益关闭。
- Premium 路线重新锁定。
- 用户可重新进入订阅页订阅。

重新订阅成功后：

- 状态重新变为 Active。
- 如果用户已使用过首月优惠，则按 $4.99/month 订阅。

## 4. 订阅状态变化逻辑

### 4.1 Free

含义：

- 用户没有有效订阅。

页面表现：

- Profile 展示 Premium 订阅引导。
- Premium 路线锁定。
- 点击 Premium 路线进入订阅页。

可进入状态：

- 订阅成功后进入 Active。

### 4.2 Active

含义：

- 用户已订阅，且当前周期未结束。

页面表现：

- Premium 路线解锁。
- 订阅页展示续订日期和付款方式。
- Profile 展示 Premium enabled。

可进入状态：

- 用户取消续订后进入 Canceled at period end。
- 续费失败后进入 Grace period。
- 当前周期结束后进入 Expired。

### 4.3 Canceled at period end

含义：

- 用户已取消自动续订，但当前周期未结束。

页面表现：

- Premium 路线仍然解锁。
- 页面展示会员可用到哪一天。
- 页面展示 Restart membership。

可进入状态：

- 用户恢复续订后回到 Active。
- 当前周期结束后进入 Expired。

### 4.4 Grace period

含义：

- 续费失败后的宽限期。
- 用户需要更新支付方式。

页面表现：

- Premium 路线仍然解锁。
- 订阅页展示支付异常提示。
- Profile 展示 Payment issue。
- 提供更新支付方式入口。

可进入状态：

- 用户更新支付方式后回到 Active。
- 宽限期结束后进入 Billing retry。

### 4.5 Billing retry

含义：

- 宽限期结束后的支付重试状态。
- Premium 权益暂停。

页面表现：

- Premium 路线锁定。
- 订阅页展示 Premium paused。
- Profile 展示 Premium paused。
- 提供更新支付方式入口。

可进入状态：

- 用户更新支付方式后回到 Active。
- 重试失败后进入 Expired。

### 4.6 Expired

含义：

- 订阅已到期，或历史退款/撤销状态被兼容转换为到期状态。

页面表现：

- Premium 路线锁定。
- 订阅页提示会员已结束。
- Profile 提示 Premium inactive。
- 用户可重新订阅。

可进入状态：

- 重新订阅成功后进入 Active。

### 4.7 Trialing

含义：

- 预留状态，用于后续免费试用扩展。

当前版本规则：

- 主流程不开放免费试用。
- Trialing 不作为有效 Premium 权益状态。

## 5. 权益判断规则

有 Premium 权益的状态：

- Active，且当前周期未结束。
- Canceled at period end，且当前周期未结束。
- Grace period，且当前周期未结束。

没有 Premium 权益的状态：

- Free。
- Trialing。
- Billing retry。
- Expired。
- 当前周期已结束的任意状态。

页面展示必须和权益判断保持一致：

- 有权益时，Premium 路线可以进入详情并开始跑步。
- 无权益时，Premium 路线保持锁定，点击后进入订阅页。

## 6. 页面行为要求

### 6.1 Profile 页

免费用户：

- 展示 Premium 订阅引导。
- 如果用户未使用首月优惠，展示首月优惠价格。
- 点击后进入订阅页。

有效会员：

- 展示 Premium enabled。
- 展示当前套餐。
- 点击后进入订阅页查看详情。

取消待生效用户：

- 展示 Ending soon。
- 展示权益保留到期日。
- 提供 Restart membership。

支付异常用户：

- Grace period 展示 Payment issue，并提示更新支付方式。
- Billing retry 展示 Premium paused，并提示更新支付方式。

过期用户：

- 展示 Premium inactive。
- 提供 Restart Premium。

### 6.2 订阅页

订阅页根据用户状态展示不同内容：

- Free：展示订阅介绍、首月优惠或常规价格、订阅按钮。
- Active：展示会员信息、续订日期、付款方式、取消入口。
- Canceled at period end：展示结束日期和恢复续订入口。
- Grace period：展示支付异常提示和更新支付方式入口。
- Billing retry：展示权益暂停提示和更新支付方式入口。
- Expired：展示重新订阅入口。

### 6.3 支付页

支付页根据场景展示不同金额：

- 首次订阅：本次应付 $0.99。
- 非首次订阅：本次应付 $4.99。
- 更新支付方式：本次应付 $0.00。

支付页提交成功后：

- 新订阅用户进入 Active。
- 支付异常用户恢复为 Active。
- 已订阅用户仅更新支付方式。

### 6.4 路线列表与路线详情

路线列表和路线详情只根据用户是否拥有 Premium 权益决定是否开放。

无权益：

- Premium 路线展示锁定状态。
- 点击 Premium 路线进入订阅页。
- 路线详情页展示 Unlock with Premium。

有权益：

- Premium 路线可进入详情。
- 路线详情页展示 Start Run。

## 7. 本地状态保存

订阅状态需要保存在本地。

要求：

- 刷新页面后，有效订阅状态必须保留。
- 用户是否使用过首月优惠必须保留。
- 本地状态损坏时，应回到 Free。
- 旧版布尔订阅标记只用于迁移或清理。
- 到期订阅在读取时应转换为 Expired。
- 历史 Refunded 状态读取时按 Expired 处理。

## 8. 验收标准

- 免费用户点击 Premium 路线后进入订阅页。
- 首次订阅用户看到首月 $0.99。
- 首次订阅成功后，Premium 路线立即解锁。
- 已使用首月优惠的用户重新订阅时，不再看到 $0.99。
- 已订阅用户可查看续订日期和付款方式。
- 已订阅用户可更新支付方式。
- 已订阅用户可取消续订。
- 取消续订后，会员权益保留到当前周期结束。
- 取消待生效用户可恢复续订。
- Grace period 用户仍可访问 Premium 路线，并看到更新订阅方式。
- Billing retry 用户不能访问 Premium 路线，并看到更新订阅方式。
- Expired 用户不能访问 Premium 路线，但可重新订阅。
- 刷新页面后订阅状态不应被清空。
- 页面文案必须与实际权益一致。
